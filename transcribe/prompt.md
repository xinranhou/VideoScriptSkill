# VideoScript Skill — 转录视频为文字脚本

## 触发方式

用户可以用以下方式触发：
- `/transcribe <视频路径>`
- `把视频转成文字脚本`
- `帮我转录这个视频`

## 执行流程

### 第一步：检查环境

在开始转录之前，调用 MCP tool `check_config`，确认：
1. ffmpeg 已安装
2. 腾讯云 ASR 凭证已配置
3. MiniMax API Key 已配置（如已配置，LLM 校正功能可用）

如果任一条件不满足，提示用户先运行 `scripts/install.sh` 完成配置。

### 第二步：获取参数

询问用户以下信息（如果用户未在命令中提供）：

1. **视频路径**（必填）：mp4 或 wav 文件路径
2. **语言**（可选，默认中文 16k_zh）：
   - 中文：`16k_zh`
   - 英文：`16k_en`
   - 粤语：`16k_zh-TW`
3. **切片时长**（可选，默认 45 秒）：30-59 之间的整数
4. **输出文件路径**（可选）：如果不指定，直接输出 Markdown 文本

### 第三步：执行转录

调用 MCP tool `transcribe_video`：

```
transcribe_video({
  "video_path": "用户提供的路径",
  "chunk_duration": 用户选择的时长（默认 45）,
  "enable_correction": true（需要 MiniMax API Key，可选）
})
```

### 第四步：处理结果

- 转录完成后，Markdown 文件会自动保存到视频同目录下
- 文件命名规则：`{视频文件名}_transcript.md`
- 直接展示 Markdown 内容给用户

### 第五步：向用户展示

将 Markdown 内容展示给用户，包含：
- 每个时间段的识别文本
- 成功/失败统计
- 文件保存路径

## 输出格式示例

```markdown
# 视频转文字脚本

## [00:00] - [00:45]
欢迎收看我今天的视频，今天我们来聊一聊...

## [00:45] - [01:30]
上一期我们讲到了...

---
*共 5 个切片，成功 4，失败 1*
```

## 错误处理

| 错误类型 | 处理方式 |
|----------|----------|
| 文件不存在 | 提示用户检查路径，可使用绝对路径 |
| ffmpeg 未安装 | 提示运行 install.sh |
| 凭证未配置 | 提示运行 install.sh |
| ASR API 失败 | 显示已成功的片段，标记失败的片段 |
| 磁盘空间不足 | 提示清理临时文件 |

## 注意事项

- 视频会自动按静音点切分为 30-59 秒片段
- 处理时长 ≈ 视频时长 × 1.2（每片段需要 ASR 调用时间）
- 凭证默认保存在 `~/.config/videoscripts/config.json`
- 支持断点续传：重复转录同一视频会跳过已成功的片段
- LLM 校正需要 MiniMax API Key（安装时配置），如未设置则跳过校正步骤
